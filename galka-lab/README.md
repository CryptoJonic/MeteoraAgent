# Galka Lab — Pool Campaign Simulator

Изолированный исторический тренажёр стратегии Galka Pool. Приложение не содержит кошелька, приватных ключей, SDK Meteora или пути отправки транзакций.

## Реализованный контракт MVP

- BTC, ETH и SOL через публичные spot-свечи Binance.
- На экране — 1h, исполнение — по 5m.
- Путь зелёной 5m-свечи: `open → low → high → close`.
- Путь красной 5m-свечи: `open → high → low → close`.
- Депозит 1,200 USDC: три пула по 400 USDC.
- В каждом пуле 40 buy bins по 10 USDC. До полного прохода они обратимы: вниз USDC превращается в asset, вверх тот же bin возвращается в USDC.
- После полного fill пул необратимо превращается в 40 sell bins.
- Sell-веса линейно растут от 1 до 4 и нормализуются к точному количеству asset.
- Исполненный sell bin удаляется; его USDC блокируется и больше не участвует в покупках.
- Касание GALKA после хотя бы одной покупки завершает кампанию: partial bins к этому моменту естественно раскрыты обратно, flipped pools заканчивают sell-лестницы, принудительно допускается закрыть только dust.
- Stop отменяет bins и продаёт остаток по текущей внутренней цене.
- Комиссии, gas, slippage, LP rewards, lending и займы равны нулю.
- Будущие свечи хранятся только внутри replay-движка. Chart получает исключительно срез до текущего cursor.
- Активный replay, конфигурации, результаты и измерения сохраняются локально.
- Свечи кэшируются в IndexedDB; загруженные интервалы учитываются отдельно.

## Termux: установка и браузерный запуск

```bash
cd ~/GalkaLab/galka-lab
pkg install nodejs-lts git -y
npm ci
npm run dev
```

Vite покажет локальный адрес. Обычно это `http://127.0.0.1:5173`.

Production-проверка:

```bash
cd ~/GalkaLab/galka-lab
npm run check
npm run preview
```

## Тесты

```bash
cd ~/GalkaLab/galka-lab
npm test
```

Тестовый набор фиксирует обязательные инварианты: независимость crosshair/GALKA/измерения, границы и капитал, обратимость partial bins, flip только при 100%, точную передачу asset в sell ladder, запрет повторной покупки settled sells, GALKA/Stop accounting, hourly replay и запрет будущих свечей.

## Capacitor Android

Конфигурация уже находится в `capacitor.config.ts`. Android-проект создаётся один раз:

```bash
cd ~/GalkaLab/galka-lab
npm run android:add
npm run android:sync
```

Для сборки APK понадобятся Android SDK/JDK. Сам симулятор и storage не зависят от Android API, поэтому тот же deterministic core работает в Vite и Capacitor WebView.

## Известные ограничения MVP

- Источник цены — Binance Spot, а не Meteora/Solana pool price.
- Внутрисвечной путь является заданной детерминированной моделью, а не tick-trade reconstruction.
- Initial window: 21 день до старта и 45 дней после; следующие блоки по 45 дней догружаются автоматически.
- GALKA фиксируется отдельной кнопкой из текущего crosshair либо точным вводом; обычное нажатие по графику её не меняет.
- Верхняя и нижняя границы имеют отдельные мобильные drag handles.
- Replay работает со скоростями x1/x5/x20/x100 в часовых свечах; x100 обрабатывается контролируемыми пакетами.
- Процентное измерение — отдельный режим и не меняет границы кампании.
- Нет условных LP-комиссий, сетевых расходов, проскальзывания и налогового учёта.
- Нет live adapter, кошелька и реальных позиций.
